# budgetApp AGENTS

## 프로젝트 설명
- 이 저장소는 CSV 기반 가계부 CLI 앱 프로젝트다.
- `data/`의 샘플 CSV를 기준으로 거래를 읽고, 추가하고, 조회하고, 요약하는 기능을 개발한다.

## 코딩 규칙
- 모든 함수와 메서드에는 타입 힌트를 반드시 작성한다.
- 함수 하나의 길이는 최대 50줄을 넘기지 않는다.

## TDD 규칙
- 구현 전에 반드시 테스트를 먼저 작성한다.
- 테스트가 실패하는 상태를 확인한 뒤 구현을 진행한다.

## 품질 규칙
- 순환복잡도는 10 이하를 유지한다.
- `radon cc` 검수 시 순환복잡도 10 초과는 실패로 간주한다.

## 품질 검수 규칙
- 커밋 전에 반드시 `qa_engineer` 서브에이전트로 품질 검수를 진행한다.
- `qa_engineer`는 변경된 파일과 관련 테스트만 기준으로 검토한다.
- 검수 결과는 문제점 또는 명시적 승인으로 반환한다.
- 검수 결과에서 지적된 항목은 반영한 뒤에만 다음 단계로 진행한다.

## 테스트 실행 명령어
- `pytest`
- `radon cc`

## 커밋 규칙
- 하나의 기능이 개발되면 커밋하고 바로 푸시한다.
- 커밋은 기능 단위로 작게 나눈다.

## 작업 원칙
- 구현 전에는 계획과 테스트를 먼저 확인한다.
- 새로운 규칙이 필요하면 이 파일을 먼저 갱신한 뒤 작업한다.
